Data Flow

  1. 1 Daily Ingestion: GitHub Actions cron fetches trials from CT.gov API (1000/page)
  2. 2 Normalization: Maps to unified schema with cancer type taxonomy
  3. 3 Classification: Claude Haiku assigns modalities & writes plain-language summaries (~$0.0002/trial)
  4. 4 Patient Matching: Claude Sonnet evaluates eligibility against user profile (~$0.10/request)

Key Design Decisions

  • ✓ CT.gov Primary: Clean JSON API, no auth, 80% coverage
  • ✓ LLM Classification: Handles intervention variations (e.g., "AdV-tk + valacyclovir + radiation")
  • ✓ Two-Stage Matching: SQL pre-filter (cheap) → LLM scoring (expensive, capped at 30)
  • ✓ Server Components: DB queries at page level, no API layer needed

Redundancy

Both have CT.gov fetcher concepts. trial-finder's is fully implemented; cancer-platform has basic sample script.

Complementary

cancer-platform's expert curation + genomic focus could extend trial-finder's matching with biomarker specificity.

Recommendation

Merge concepts: Use trial-finder as base, add cancer-platform's curation queue + biomarker features as v2.
